# デスクトップのリストシェブロン表示設計

## 目的

デスクトップの Live Preview では、折りたたみ可能なリスト行へマウスポインタを重ねたときだけ、その行のネイティブシェブロンを表示する。

カーソルが置かれたアクティブ行であっても、マウスポインタが行外にあるときは表示しない。

縦線の折りたたみ操作が有効な場合も同じ表示条件を使い、シェブロンと縦線の両方から折りたたみを操作できるようにする。

## 対象範囲

対象は、デスクトップの `.markdown-source-view.mod-cm6.is-live-preview` にあるリスト行のネイティブ `.collapse-indicator` だけとする。

ネイティブシェブロンが存在しない leaf 行には、新しい要素を追加しない。

見出しのシェブロン、Reading View、モバイルの右端折りたたみコントロールは変更しない。

## 表示条件

通常時のリストシェブロンには `visibility: hidden`、`opacity: 0`、`pointer-events: none` を適用する。

対象の `.cm-line.HyperMD-list-line` 自体が `:hover` のときだけ、同じシェブロンを表示してクリック可能にする。

表示判定には editor selection、`.cm-active`、キーボードフォーカス、縦線操作の設定を使わない。

表示状態を CSS だけで決めるため、イベントリスナー、DOM marker、タイマー、座標キャッシュは追加しない。

## 縦線との間隔

Obsidian 1.13.2 の既定配置では、隣接する縦線の間隔が18px、シェブロンの SVG が10pxであり、SVG の内側端が現在のガイド位置と一致していた。

シェブロンの操作領域を、外側のガイドと現在のガイドの間へ移す。

操作領域の開始位置と幅は `--list-indent` から計算し、隣接するガイド間の全幅へ連続させる。

操作領域内では SVG を中央揃えにし、ネイティブ SVG 自体の transform は変更しない。

これにより、既定値では18pxの操作領域内に10pxの SVG が収まり、両側のガイドから4pxずつ離れる。

操作領域を行の本文まで連続させるため、最外層の行でも本文からシェブロンへポインタを移す途中で `:hover` が途切れない。

CSS 変数が未定義のテーマでも成立するよう、`--list-indent` には18pxの fallback を設ける。

背景色でガイドを覆ったり、縦線の位置、太さ、色、z-indexを変更したりしない。

最外層では18px幅の outer guide が通常の fold indicator より前面にあるため、そのままでは表示したシェブロンをクリックできない。

デスクトップで縦線操作が有効かつ行がホバーされている間だけ `.cm-fold-indicator` を outer guide より前面へ出し、18px幅の `.collapse-indicator` 自体は `pointer-events: none`、中央の10px SVGだけは `pointer-events: auto` にする。

これにより、シェブロン中央ではネイティブfoldingを使え、左右4pxでは既存のouter guideまたはnative indent guideを操作できる。

## 既存機能との関係

縦線操作が有効な間にシェブロンを常時隠している既存ルールは削除する。

新しいデスクトップ用ルールは `body:not(.is-mobile)` へ限定し、モバイル用 `.bullet-plugin-mobile-right-fold-controls` の常時表示と右端配置へ影響させない。

操作には既存のネイティブ `.collapse-indicator` を使い、Obsidian の fold transactionを置き換えない。

シェブロンを非表示にしている間は hit target も無効にし、行ホバーと縦線ホバーの判定を妨げない。

縦線操作が無効な場合は18pxのネイティブ操作領域全体をクリック可能にし、本文から操作領域までホバーを連続させる。

## テスト

CSS contract testでは、次の条件を検証する。

- デスクトップの Live Preview リスト行だけを対象にする。
- 通常時は `visibility`、`opacity`、`pointer-events` によって非表示にする。
- 行の `:hover` だけで表示と hit targetを復元する。
- ガイド間の操作領域を `--list-indent` から計算し、SVG を中央揃えにする。
- 縦線操作中はfold indicatorの重なり順をホバー中だけ上げ、SVGだけをネイティブのhit targetにする。
- 見出しとモバイルへ作用する無限定な selectorを追加しない。
- 縦線操作の body classに依存した常時非表示ルールを残さない。

実 Obsidian では、リポジトリ内の `vault` を使って次を確認する。

- 行外、editor selectionだけがある行、leaf 行ではシェブロンが表示されない。
- rootと入れ子のfoldable行では、行の本文上へポインタを置いたときだけシェブロンが表示される。
- 表示された SVG と両側の縦線の間に正の間隔があり、本文から操作領域までホバーが途切れない。
- 縦線操作中の `elementFromPoint()` はSVG中央でネイティブアイコンを返し、左右の余白で対応するガイドを返す。
- シェブロンと縦線のどちらからでもfoldとunfoldが動作する。
- モバイルでは既存の右端コントロールが常時表示される。

## 完了条件

デスクトップのリストシェブロンは、折りたたみ可能な行へマウスポインタを重ねたときだけ表示される。

rootと入れ子のどちらでも、シェブロンのアイコンが隣接する縦線へ重ならない。

見出し、モバイル、ネイティブのfolding、縦線のfoldingに回帰がない。
